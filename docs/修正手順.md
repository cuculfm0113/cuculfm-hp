## 2. ローカル開発

### 2.1 リポジトリのクローン（初回のみ）

```bash
git clone https://github.com/（ユーザー名）/（リポジトリ名）.git
cd （リポジトリ名）
```

### 2.2 作業ブランチの作成（任意）

```bash
git checkout -b fix/（修正内容の簡潔な説明）
```

### 2.3 編集後の確認

ブラウザで直接HTMLを開くか、簡易サーバーで確認：

```bash
# Python 3の場合
python3 -m http.server 8000

# ブラウザで http://localhost:8000 を開く
```

## 3. 変更の反映手順

### 3.1 ファイルの編集

エディタ（VS Code、Cursor等）で該当ファイルを編集する。

### 3.2 Gitでのコミット

```bash
# 変更内容を確認
git status

# 変更ファイルをステージング
git add .
# または特定ファイルのみ
git add index.html style.css

# コミット
git commit -m "修正内容の簡潔な説明（例：トップページの文言を更新）"
```

### 3.3 プッシュとNetlifyデプロイ

```bash
git push origin main
```

NetlifyはGitHubと連携しているため、`git push` すると自動でビルド・デプロイが実行されます。
数分後に https://cucul-fm.com で変更が反映されます。

## 4. よくある修正パターン

### 4.1 テキスト・文言の変更

- **トップページ**: `index.html` を編集
- **事業ページ**: `services/（事業名）/index.html` を編集
- **採用ページ**: `recruit/index.html` を編集
- **ブログ・記事**: `blog/` または `articles/` 内の該当HTML

### 4.2 画像の差し替え

1. 新しい画像を適切なフォルダに配置（例: `images/`）
2. HTML内の `src="..."` を新しい画像パスに変更
3. 推奨: ファイル名は小文字・ハイフン区切り（例: `new-banner.png`）

### 4.3 ニュース提示板の枠にテキスト追加

`index.html` 内の `.bulletin-card-placeholder` を編集。
枠3・枠4の「Coming Soon」部分を、以下と同様の構造に変更：

```html
<div class="bulletin-card bulletin-card-link">
  <div class="bulletin-card-inner">
    <span class="bulletin-badge">バッジ文字</span>
    <h3 class="bulletin-card-title">タイトル</h3>
    <p class="bulletin-card-desc">説明文</p>
  </div>
</div>
```

### 4.4 新規ページの追加

1. 新規HTMLファイルを作成
2. 既存ページ（`index.html` や `services/construction/index.html` など）の構造をコピーし、内容を変更
3. メタタグ（`<title>`, `<meta name="description">`）を設定
4. 他ページからのリンクを追加
5. **sitemap.xml を更新**（下記5.2参照）

## 5. SEO・メタ情報の更新

### 5.1 ページタイトル・description の変更

各HTMLの `<head>` 内を編集：

```html
<title>ページタイトル | CUCUL FM</title>
<meta name="description" content="ページの説明文（120〜160文字程度）">
```

### 5.2 sitemap.xml の更新

新規ページを追加した場合、`sitemap.xml` にURLを追加する。

形式：

```xml
<url>
  <loc>https://cucul-fm.com/（パス）/</loc>
  <lastmod>YYYY-MM-DD</lastmod>
  <changefreq>monthly</changefreq>
  <priority>0.8</priority>
</url>
```

- `lastmod`: 最終更新日（今日の日付）
- `changefreq`: `weekly`（更新頻繁）/ `monthly`（普通）/ `yearly`（ほぼ変わらない）
- `priority`: `1.0`（トップ）/ `0.9`（重要）/ `0.8`（その他）

### 5.3 robots.txt

通常は編集不要。クロールを禁止したいパスがある場合のみ `robots.txt` を編集。

## 6. トラブルシューティング

### 6.1 デプロイが失敗する

- **Netlify ダッシュボード** → Deploys でエラーログを確認
- よくある原因: 不正なHTML、パスの typo、存在しないファイルへの参照
- `git status` で意図しないファイルが含まれていないか確認

### 6.2 画像が表示されない

- ファイルパスが正しいか確認（大文字小文字、スラッシュの有無）
- 画像ファイルがリポジトリにコミットされているか確認
- パスはプロジェクトルートからの相対パス（例: `images/photo.jpg`）

### 6.3 404ページが表示されない

- `404.html` がプロジェクトルートにあるか確認
- Netlifyは自動で `404.html` を404エラー用に使用する

### 6.4 スタイルが崩れる

- `style.css` へのパスが正しいか確認（サブフォルダのページは `../style.css` など）
- ブラウザのキャッシュをクリアして再読み込み
